    Bootstrap and Provisioning

    Here is an outline of the bootstrap and provisioning process:

    • cloud-start
      • launch instance
        • calls launch_new_instance! on remoter_base (ec2, vmware, etc.)
        • waits for an ip to be assigned
        • waits for ssh availability
      • cloud-provision
        • bootstrap
          • compiles all necessary dependencies for bootstrap
          • rsync's the dependencies
          • runs array of commands that bootstraps the machine
        • configure
          • compiles and packages compilation of cookbooks and recipes
          • rsync's the package
          • runs dependency_resolver (chef/puppet)
          • runs verify blocks to verify the cloud has launched and passes verification
            • reruns configure if fails
