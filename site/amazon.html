    Terminology

    For PoolParty and general cloud computing terminology Check here
    EC2
    A platform for launching machine instances
    AMI
    Amazon Machine Image. Conceptually similar to a Xen or VMware image. There are hundreds of pre-built AMI's for you to choose from. PoolParty uses ec2ubuntu.
    S3
    Simple Storage Service (S3). Persistent data storage. Data is stored in 'buckets'. EC2 does not use S3 directly
    EBS
    Elastic Block Store. Persistent disks for EC2 instances. Can store snapshots on S3. Its fast but only mounts to one instance at a time. i.e. It is not a SAN.
    Elastic IP
    Typical EC2 instance IPs are dynamic and assigned at boot time. Elastic IPs provide a way for you to assign static IPs to nodes.
    Security Groups
    Amazon has a strict firewall between the world and your nodes and between nodes within EC2. Each instance is assigned to one (or more?) security group. Each security group has its own set of security policies. These policies can be controlled by using the ec2-authorize binary. In the default security group, all nodes have unlimited (network) access to each other. Everything else is turned off. (This includes port 22 for ssh and port 80 for http )
    Cloud Front
    Amazon's pay-as-you-go "self service" CDN. You upload files to S3 and then they are available through public URLs.
